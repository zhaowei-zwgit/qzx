# Polyp Training and Evaluation Design

## Goal

Provide a complete, reproducible training and evaluation workflow for the
prepared Kvasir and CVC-ClinicDB polyp segmentation data, and verify it with a
small smoke test suitable for the local 4 GB GPU.

## Command Interface

The workflow exposes one module with three commands:

```powershell
python -m sam2unet.experiment train --config configs/polyp_train.json
python -m sam2unet.experiment evaluate --config configs/polyp_train.json --checkpoint PATH
python -m sam2unet.experiment smoke --config configs/polyp_train.json
```

Common command-line overrides select the bridge mode, device, epoch count,
sample limits, output directory, and resume checkpoint.

## Dataset Pipeline

A manifest dataset reads the normalized CSV files generated by
`sam2unet.data`. Images are converted to RGB, masks are converted to binary
single-channel tensors, and both are resized to the configured input size.

Training applies synchronized horizontal and vertical flips. Evaluation is
deterministic and applies no augmentation. Manifest-relative paths are resolved
from the project root so commands do not depend on the current working
directory.

## Loss and Metrics

Training uses deep supervision across all three model logits:

```text
loss = sum(BCEWithLogits(logit, mask) + soft IoU loss(logit, mask))
```

Evaluation uses the primary output only and reports dataset-level mean Dice,
IoU, and MAE after sigmoid activation. Dice and IoU use threshold 0.5.

## Training Runtime

The training loop supports:

- AdamW and cosine learning-rate decay.
- Automatic mixed precision when CUDA is active.
- Batch size 1 and gradient accumulation for the local 4 GB GPU.
- Gradient clipping.
- Reproducible random seeds.
- Per-epoch evaluation on Kvasir and CVC-ClinicDB.
- `latest.pt` and `best.pt` checkpoints.
- Resume state containing model, optimizer, scheduler, scaler, epoch, best
  score, and history.
- JSON history and final evaluation summaries.

The best checkpoint is selected by the average Dice across configured test
sets.

## Model Construction

Normal training builds `SAM2UNetFusion` with the configured SAM2 Hiera-L
checkpoint and one of `rfb`, `static`, or `full`.

The smoke command uses a small injected four-level encoder and reduced feature
channels. It executes a complete train-save-resume-evaluate cycle on a limited
number of real polyp samples without requiring the Hiera-L memory footprint.
It then separately attempts one real SAM2 Hiera-L forward pass. A real-forward
failure caused by the local environment or 4 GB memory limit is recorded in a
JSON report and does not invalidate the workflow smoke test.

## Error Handling

Commands fail clearly when manifests, SAM2 source, or checkpoints are missing.
Training refuses empty datasets and invalid gradient-accumulation values.
Evaluation refuses checkpoints without a model state dictionary.

## Testing

Automated tests cover:

- Manifest image/mask loading and synchronized transformations.
- Soft IoU and deep-supervision loss behavior.
- Dice, IoU, and MAE accumulation.
- One training epoch updating a lightweight model.
- Checkpoint save/resume state.
- Evaluation result structure.
- Smoke command output artifacts.

The full project test suite and syntax compilation must pass before completion.
